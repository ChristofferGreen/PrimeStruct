SoA de-builtinization roadmap note: the target state for `soa_vector` is also a stdlib `.prime` implementation. 1. Finish only the runtime/ownership substrate that self-contained `.prime` containers genuinely need.

Map-first rollout note: `map` should be deleted from compiler-owned code first because experimental `map` already has a real `.prime` storage/runtime implementation; `vector` should follow once the remaining ownership/drop substrate is in place.
